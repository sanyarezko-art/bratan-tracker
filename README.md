# БРАТАН-трекер

Анонимный браузерный торрент-трекер/каталог на GitHub Pages.
Всё — клиент, «трекер», передача данных — работает прямо в браузере через
[WebTorrent](https://webtorrent.io) и WebRTC.
Сервера нет. Бэкенда нет. Аналитики нет. Логов нет.

Живая страница: **https://sanyarezko-art.github.io/bratan-tracker/**

## Что умеет

- **Открыть** магнет-ссылку → файлы появляются в браузере, видео/аудио/картинки
  смотрятся прямо на странице, остальное сохраняется на диск кнопкой.
- **Создать** торрент из любых файлов (drag-and-drop) → клиент сам считает хэши,
  начинает сидировать и отдаёт magnet + share-ссылку + QR-код.
- **Поделиться** через share-ссылку вида
  `https://sanyarezko-art.github.io/bratan-tracker/#magnet=<urlencoded-magnet>`.
  Кто перейдёт — у того сразу начнётся скачивание.
- **Каталог** магнет-ссылок из `catalog.json` с поиском. Добавляется через PR.
- **Глобальная статистика** (пиры, скорости, ratio) и прогресс по каждому
  торренту.

## Как устроено (кратко)

- `index.html`, `styles.css`, `app.js` — статическая страница.
- Внешние скрипты — только [`webtorrent`](https://www.npmjs.com/package/webtorrent)
  и [`qrcode-generator`](https://www.npmjs.com/package/qrcode-generator), оба
  подгружаются с [jsDelivr](https://www.jsdelivr.com/) и пришпилены по
  Subresource Integrity.
- Никаких куков, `localStorage`, сторонних шрифтов, analytics.
- `Content-Security-Policy` разрешает только `self` + `cdn.jsdelivr.net` для
  скриптов. `connect-src *` оставлен для WebSocket-трекеров и WebRTC, иначе
  P2P не заведётся.
- Трекеры по умолчанию: `wss://tracker.openwebtorrent.com`,
  `wss://tracker.webtorrent.dev`, `wss://tracker.btorrent.xyz`,
  `wss://tracker.files.fm:7073/announce`.

## Безопасность и анонимность

Прочитай внимательно перед использованием:

- **IP виден пирам.** Как и любой торрент-клиент, WebRTC открывает прямые
  соединения к пирам — твой публичный IP видит каждый пир, к которому
  подключается браузер. Если это проблема — используй VPN или запускай
  браузер через Tor **до** открытия сайта.
- **Сайт ничего не хранит про тебя.** Страница статическая, никаких
  серверных логов нет. Но пиры и третьи стороны, через которые идёт трафик
  (в т.ч. твой провайдер), могут видеть соединения.
- **Трафик не шифруется между пирами сильнее, чем DTLS у WebRTC.** Для
  защиты от MITM-анализа это тебе не «end-to-end privacy tool».
- **Куски торрента живут в памяти вкладки.** Закрыл вкладку — раздача
  остановлена, данные пропали (если явно не сохранил на диск).
- **Соблюдай закон и авторские права.** Этот проект — инфраструктура; за
  содержимое раздач отвечает тот, кто их создал.

## Локальная разработка

Никакой сборки не нужно. Открой `index.html` через любой локальный
http-сервер (например, `python3 -m http.server 8080`) и ходи на
`http://localhost:8080/`. Через `file://` WebTorrent и `fetch('catalog.json')`
работать не будут.

## Как добавить в каталог

Смотри [`CONTRIBUTING.md`](./CONTRIBUTING.md). Если коротко — PR в
`catalog.json` с валидным magnet-URI.

## Деплой

`main` → GitHub Pages через `.github/workflows/pages.yml`. Каждый коммит в
`main` публикуется автоматически.
